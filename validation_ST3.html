Validation Report

This page showcases the HTML and CSS validation results for the pages developed as part of the Zero Hunger Sri Lanka website project. All pages have been validated using the W3C Markup Validation Service and W3C CSS Validation Service.

Feedback Page Validation

HTML Validation

The Feedback page HTML has been validated using the W3C HTML Validator. The page passed validation without any errors or warnings, confirming it complies with HTML5 standards.

CSS Validation

The CSS for the Feedback page has been validated using the W3C CSS Validator. The stylesheet passed validation with no errors, confirming proper CSS syntax and standards compliance.

Team Page Validation

HTML Validation

The Team page also passed validation successfully with no errors. The semantic HTML structure was particularly praised by the validator for its accessibility features.

CSS Validation

The Team page CSS was validated without errors. The validator confirmed proper use of CSS properties and selectors throughout the stylesheet.

Content Page Validation

HTML Validation

The Content page is error-free and complies with W3C standards. The validator confirmed proper use of HTML5 elements and attributes throughout the page.

CSS Validation

The Content page stylesheet passed CSS validation with no errors. All styles were properly implemented according to CSS specifications.

Reflection

Through the validation process, I gained valuable insights into writing clean, standards-compliant HTML and CSS code. Initially, I encountered some minor issues with improper nesting of elements and missing alt attributes in HTML, and some vendor prefix issues in CSS.

The validation tools served as excellent quality control mechanisms, catching issues that might have otherwise gone unnoticed. For CSS specifically, I learned the importance of:

  • Proper property ordering for better maintainability
  • Using standardized properties instead of browser-specific prefixes
  • Organizing stylesheets logically
  • Optimizing selector specificity

This experience reinforced the importance of semantic HTML and clean CSS. Moving forward, I plan to validate both HTML and CSS regularly during development rather than just at the end, to catch issues early and maintain higher coding standards throughout the development process.

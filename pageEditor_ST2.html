My Role and Completed Tasks

Role: As Student 2, I was responsible for designing and implementing the following core components of our group website:

  • Home Page (home.html)
  • Global CSS Template (global.css)
  • Table Page (table.html)
  • Content Page: Causes & Facts (content_ST2.html)

Technical Descriptions

Home Page

  • The home page sets the foundation for the user’s experience across the site. I used a clean, responsive layout built with semantic HTML5 elements such as <header>, <nav>, and <main> to ensure accessibility and structure. The navigation bar was created using a <ul> and styled with display: flex to align links horizontally across the top of the page.

  • I applied a global CSS template to enforce consistency across the entire website. For example, I used the Google Fonts Poppins and Roboto for headings and paragraph text respectively, to create a modern, legible appearance. Hover effects were added using the :hover pseudo-class on links (a:hover) to give interactive feedback when users move their cursor over the navigation menu.

  • The logo and site title are aligned using flexbox, and spacing is managed using margin and padding utilities. To ensure responsiveness on different screen sizes, I used max-width and media queries in the CSS file.


    • Key Tags & Properties

    1. HTML: <header>, <nav>, <ul>, <li>, <img>, <a>, <main>
    2. CSS: display: flex, gap, :hover, text-align, font-family, media queries

    Table Page

  • The table page is one of the most interactive parts of the site and displays a region-wise comparison of hunger impacts across Sri Lanka. I structured the data using the <table> element with headers in <thead>, and grouped data rows under <tbody>. To create a visually organized structure, I used rowspan and colspan to merge header cells and separate regions (Urban, Estate, Coastal) under unified rows.

  • Thumbnails were embedded within the table header using <img> tags. These icons were wrapped in <th> elements and enhanced with a tooltip feature using CSS only — this was achieved using a custom data-tooltip attribute and ::after pseudo-element styling on hover.

  • To improve readability, I styled alternating rows with tr:nth-child(even) for zebra striping. A box-shadow was applied to the entire table wrapper for a modern, elevated look. Hover effects (tr:hover) give visual feedback and bolden the row, while icons such as ✓ and ⚠️ were added inside <td> elements to show food security visually.

  • I also included clickable links within each row using anchor tags <a> with target="_blank" to open full reports (like the IJRAR PDF) in a new tab. One of the cells uses colspan="5" to span a row across the entire table for grouping.


    • Key Tags & Properties

    1. HTML: <table>, <thead>, <tbody>, <th>, <td>, <img>, rowspan, colspan
    2. CSS: nth-child(even), :hover, box-shadow, border, padding, position: relative, ::after (for tooltips)

    Content Page

  • This page presents the “Causes and Facts of Hunger in Sri Lanka.” I used a multi-section layout with internal linking to allow users to jump between facts, causes, videos, and sources. Headings (<h2>, <h3>) were logically organized and wrapped in <section> elements with id attributes to enable anchor navigation.

  • The facts section was implemented using a responsive CSS grid (display: grid) with grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)) to ensure it adapts to any screen width. Each fact was presented inside a div styled as a card using box-shadow, border-radius, and hover animations (transform: translateY(-5px)), creating an engaging experience.

  • For the causes section, I designed a different layout to distinguish it visually. I used vertically stacked “info blocks” with section titles, emojis, and paragraph explanations. This helped maintain contrast between visual layouts and improved scannability.

  • To meet accessibility and interactivity requirements, I added:
    1. A fixed “Go to Top” button (position: fixed) styled with a circular background and smooth hover transitions
    2. YouTube videos embedded using <iframe> with responsive styling
    3. All images included alt text and were styled with border-radius, margin, and width: 100% to keep them responsive


    Key Tags & Properties

    1. HTML: <section>, <h2>, <h3>, <a>, <iframe>, <img>, <div id="causes">
    2. CSS: display: grid, gap, box-shadow, border-radius, position: fixed, scroll-behavior: smooth, transform, :hover

    Challenges and Lessons Learned

  • One of the main challenges I faced was merging table headers using rowspan while keeping the table visually balanced on mobile screens. I resolved this by adjusting column widths and padding. Another challenge was ensuring that internal links on the content page scrolled to the correct position across different browsers — I fixed this by testing anchor IDs and using scroll-behavior: smooth in CSS.

  • From these experiences, I learned how to structure semantic, accessible pages and how to make styling decisions that balance readability with aesthetic value.

    • Accessibility

    All pages use semantic HTML tags like <header>, <nav>, <footer>, and <section>. Images include meaningful alt text, and interactive elements like buttons and links use descriptive title attributes. The "Go to Top" button is always visible and keyboard-navigable. Color contrast was checked to support readability.

    Compliance

    All my pages follow best practices aligned with Janet web publishing guidelines. I used valid, semantic HTML structure and avoided deprecated tags. CSS is separated where appropriate, and inline styles are minimal. External PDFs are opened in new tabs, and file naming conventions follow web-safe characters and lowercase patterns.
